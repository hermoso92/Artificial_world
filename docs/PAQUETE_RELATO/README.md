# Paquete de relato — Artificial World

Documentación para explicar el proyecto a dirección, familia, inversores y público no técnico sin romper la verdad técnica del repo.

## Regla del paquete

Este paquete no existe para inflar Artificial World.
Existe para darle una voz potente sin perder trazabilidad.

Toda pieza de relato debe respetar la separación entre:

- `real`
- `demo`
- `defendible`
- `vision`
- `roadmap`

Documento rector para esa separación:

- [docs/REALIDAD_VS_VISION.md](../REALIDAD_VS_VISION.md)

## Documento único

Para una versión consolidada de todo el relato en un solo archivo: [docs/DOCUMENTACION_COMPLETA.md](../DOCUMENTACION_COMPLETA.md)

## Convenciones de mensaje

- **Nombre del producto:** Artificial World.
- **Artificial Word:** nombre heredado que puede aparecer en docs antiguos; evitarlo en material nuevo.
- **DobackSoft:** vertical aplicada; en este repo es demo; producto B2B en repo separado.
- **Valor histórico del proyecto:** persistencia, memoria, trazabilidad y coste cero por decisión.
- **Nueva dirección estratégica:** sistema local y auditable de inteligencias coordinadas para comprender proyectos reales.
- **Regla editorial:** la narrativa nace de la evidencia y no al revés.

## Qué sí puede hacer este paquete

- amplificar la ambición del proyecto
- traducir arquitectura en historia comprensible
- presentar Artificial World como una categoría en formación
- sostener una voz de producto fuerte

## Qué no puede hacer este paquete

- fingir que la visión futura ya está implementada
- borrar la diferencia entre motor Python, demo web y verticales
- usar claims técnicos sin prueba o sin contexto
- vender integración o escala no demostradas
