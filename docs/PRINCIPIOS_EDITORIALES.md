# Principios editoriales — Artificial World

## Constitución editorial mínima

1. Ningún benchmark sin prueba reproducible.
2. Ninguna promesa técnica sin fuente.
3. Nunca mezclar demo con producto real.
4. Nunca vender visión como estado actual.
5. Nunca usar IA como sustituto de arquitectura.
6. Toda afirmación importante debe clasificarse como `real`, `demo`, `defendible`, `vision` o `roadmap`.
7. Toda contradicción relevante debe mantenerse visible hasta resolverse.
8. Toda narrativa pública debe nacer de evidencia técnica suficiente.
9. Si una frase suena mejor de lo que puede demostrarse, se corrige o se elimina.
10. El tono puede ser épico. La base debe ser auditable.

## Reglas de uso

### Claims

- todo claim técnico debe poder rastrearse a código, documentación o prueba
- todo claim estratégico debe indicar si describe presente o dirección futura
- todo claim comparativo debe decir contra qué se compara

### Demos

- una demo nunca debe presentarse como producto completo
- una vertical nunca debe absorber la identidad de la plataforma base
- una integración parcial nunca debe narrarse como integración cerrada

### Visión

- la visión puede ser grande, pero no debe invadir la zona de realidad actual
- si algo todavía depende de contratos, integraciones o benchmarks futuros, se marca como visión o roadmap

### Documentación viva

- la documentación debe actualizarse cuando cambie la verdad del sistema
- una fuente antigua no debe imponerse sobre evidencia nueva
- si dos documentos se contradicen, la contradicción debe registrarse

## Test editorial

Antes de publicar una frase importante, hacer tres preguntas:

1. ¿Se puede demostrar?
2. ¿Se entiende en qué nivel está: real, demo, defendible, visión o roadmap?
3. ¿Sobrevive a una lectura hostil sin parecer humo?

Si la respuesta no es sí en las tres, la frase todavía no está lista.
