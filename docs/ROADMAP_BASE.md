# Roadmap base — Artificial World

## Fase 0 — Verdad actual

Objetivo:
fijar la línea base real del proyecto.

Entregables:

- README alineado con la realidad actual
- mapa claro de `real`, `demo`, `defendible`, `vision` y `roadmap`
- narrativa base y principios editoriales

Riesgos:

- arrastrar claims heredados inconsistentes
- mezclar la historia anterior del proyecto con la nueva tesis sin jerarquía

Criterio de éxito:

- una persona nueva entiende el proyecto en menos de 5 minutos
- el repo deja claro qué existe hoy y qué no

## Fase 1 — Base operativa

Objetivo:
convertir el análisis local en un flujo repetible.

Entregables:

- ingestión de repositorios
- índice estructural del proyecto
- expediente vivo inicial
- capa básica de seguridad y permisos

Riesgos:

- meter demasiada complejidad demasiado pronto
- diseñar multiagente antes de cerrar un buen expediente base

Criterio de éxito:

- el sistema puede producir un expediente coherente sobre un repo local

## Fase 2 — Comparación de repositorios

Objetivo:
hacer útil el contraste externo.

Entregables:

- selección de referencias aprobadas
- motor comparativo
- matriz de diferencias
- re-comparación versionada

Riesgos:

- comparaciones superficiales
- referencias mal elegidas
- exceso de opinión con poca evidencia

Criterio de éxito:

- el sistema detecta diferencias accionables entre un repo y al menos dos referencias

## Fase 3 — Coordinación de agentes

Objetivo:
pasar de un pipeline lineal a una orquestación especializada.

Entregables:

- taxonomía inicial de agentes
- contratos de input y output
- políticas de activación
- auditoría transversal del flujo

Riesgos:

- agentes redundantes
- salidas duplicadas
- más complejidad que claridad

Criterio de éxito:

- cada salida muestra qué agente participó y con qué función

## Fase 4 — Experiencia fundador

Objetivo:
empaquetar el valor real para adopción temprana.

Entregables:

- modo fundador
- onboarding inicial
- plantillas de análisis y comparativas
- panel o expediente reutilizable por proyecto

Riesgos:

- vender antes de pulir
- prometer acceso especial sin mecanismo real
- soporte manual excesivo

Criterio de éxito:

- un fundador obtiene valor claro desde su primer caso real

## Fase 5 — Comunidades de agentes

Objetivo:
abrir el sistema a ecosistemas más amplios sin perder trazabilidad.

Entregables:

- bibliotecas de agentes o perfiles de trabajo
- referencias compartidas
- mecanismos de extensión controlados
- comunidad open source orientada a utilidad real

Riesgos:

- fragmentación
- extensiones de baja calidad
- deriva narrativa hacia humo de plataforma total

Criterio de éxito:

- terceros pueden extender Artificial World sin romper su disciplina editorial ni su trazabilidad
