SECCIÓN 4 — ARTIFICIAL WORLD (producto completo)

Tag: "Producto 1 — Motor de simulación"

Título: "Artificial World"

Subtítulo: "Motor de civilizaciones emergentes con agentes autónomos basados en utilidad. Sin LLMs. Comportamiento emergente real."

Descripción técnica:

Motor Python 2D con función de utilidad multidimensional (13 tipos de acción, 9 modificadores). Los agentes deciden solos en cada tick: explorar, comer, compartir, robar, huir, atacar, descansar... La decisión es calculable, reproducible y auditable. Memoria espacial y social persistente. Modo Sombra para contrafácticos. Watchdog de vigilancia activa. 68+ tests. CI en GitHub.

Fórmula central: 

U(a, i, t) = U_base(a) + Σ δk(a, i, t) + ε(i, t)

Donde δk son los 9 modificadores: mod_hambre, mod_energia, mod_riesgo, mod_rasgo, mod_anti_oscilacion, mod_memoria, mod_relaciones, mod_directivas, mod_autonomia.

Features (6 cards):

|-------|---------|---------|

| 🏛️ | 9 Semillas de Civilización | tribu fronteriza, colonia tecnócrata, comunidad espiritual, reino guerrero, ciudad comerciante, colonia paranoica, imperio decadente |

| 🛡️ | Refugios con Memoria | unidad base con recursos, moral, seguridad y memoria local |

| ⚔️ | 13 Acciones Autónomas | COMER, EXPLORAR, HUIR, ROBAR, COMPARTIR, ATACAR, MOVER, DESCANSAR, SEGUIR, EVITAR, IR_REFUGIO, RECOGER_COMIDA, RECOGER_MATERIAL |

| 🔬 | Función de Utilidad | U(a,i,t) = U_base + Σ modificadores + ε determinista |

| 📜 | Crónica Fundacional | JSON + Markdown + PDF generados automáticamente desde el runtime |

| 👁️ | Modo Sombra | simulación especular para evaluar contrafácticos sin modificar el mundo |

SECCIÓN 9 — SISTEMA CHESS (AUDITORÍA INDEPENDIENTE)

Tag: "Auditoría independiente"

Título: "El Sistema Chess"

Subtítulo: "Un espejo que no puede mentir."

Descripción:

6 agentes dockerizados en red aislada, con el repositorio montado en READ-ONLY. Sin acceso a producción. Sin estado entre ejecuciones. Sin posibilidad de ser influenciados por los creadores. Cada uno analiza una especialidad diferente y reporta al coordinator. El coordinator genera REPORTE_CHESS_1.md.

Tabla de agentes:

| Agente | Especialidad | Verifica |

|--------|-------------|---------|

| agent-docs | Documentación | Claims prohibidos, enlaces rotos |

| agent-backend | API y seguridad | console.log, URLs hardcodeadas, catches vacíos |

| agent-frontend | React | Componentes >300 líneas, sin alt en imágenes |

| agent-bd | Base de datos | SQL injection, SELECT *, sin IF NOT EXISTS |

| agent-tests | Cobertura | Tests sin assertions, ratio <30% |

| agent-marketing | Narrativa | Claims sin evidencia, overselling |

"Los creadores no pueden auditarse a sí mismos con objetividad. La solución es un sistema completamente separado, sin acceso de escritura, que analiza el mismo repositorio desde fuera."

REFLEXIÓN FILOSÓFICA — ¿Estamos en el camino adecuado?

Lo que está absolutamente bien

El núcleo técnico es defendible. La función de utilidad es transparente. Los tests existen. La taxonomía REAL/DEMO/ROADMAP es una de las ideas más poderosas del proyecto porque convierte la honestidad en diferencial competitivo. El Sistema Chess es conceptualmente brillante: institucionaliza la separación creador/auditor que la industria raramente formaliza.

Lo que se puede mejorar

El riesgo más grande no es técnico: es la dispersión narrativa. Tienes Artificial World, DobackSoft, FireSimulator, los juegos, el paper, el portfolio, el Sistema Chess, HeroRefuge... cada uno con su identidad. Eso es riqueza real, pero para una persona externa es demasiado para procesar sin una jerarquía visual brutal.

La segunda mejora posible: la separación motor Python/web tiene un coste narrativo. Para inversores o clientes, "dos motores que no hablan entre sí" suena a deuda técnica aunque sea una decisión deliberada. La forma de defenderse ya está en el paper, pero el sitio web debe comunicarla como fortaleza.

La tercera: la sesión canónica (semilla 42, 200 ticks) es un activo enorme. Un video de 90 segundos mostrándola en tiempo real valdría más que las 10 primeras secciones de cualquier web. Es el punto donde el sistema deja de ser abstracto y se convierte en algo que la gente puede ver moverse.

El camino correcto

Mantener exactamente este nivel de honestidad técnica. Añadir el video. Reducir la dispersión narrativa con una jerarquía visual clara. Y publicar el paper en arXiv o una preprint server: un DOI convierte cualquier afirmación en cita verificable, lo cual es exactamente lo que el proyecto defiende como principio.
