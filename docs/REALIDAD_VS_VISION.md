# Realidad vs vision — Artificial World

## Propósito

Este documento existe para impedir que Artificial World se sobreexplique, se sobreprometa o se contradiga.

**Ownership:** DobackSoft = producto principal. Artificial World = laboratorio hermana. Juego = superficie de demo. Ver [docs/OWNERSHIP_ESTRATEGICO.md](OWNERSHIP_ESTRATEGICO.md).

Antes de publicar una claim, redactar una landing, preparar un vídeo o presentar el proyecto, hay que pasar por aquí.

## Regla base

Toda afirmación importante debe caer en una de estas cinco categorías:

- `real`
- `demo`
- `defendible`
- `vision`
- `roadmap`

Si no sabemos en qué categoría está, todavía no está lista para salir.

## Tabla maestra

| Tema | Estado | Qué sí puede decirse | Qué no debe decirse todavía |
|------|--------|----------------------|-----------------------------|
| Motor Python | Real | Existe un motor local con memoria, persistencia, decisión por utilidad y Modo Sombra | Que ya es una plataforma total de agentes |
| Web fullstack | Demo | Existe una demo funcional con backend y frontend propios | Que la web es la interfaz directa del motor Python |
| IA local base | Real pero acotada | Hay una base local para chat, resumen y análisis puntuales | Que ya existe un sistema multiagente maduro |
| DobackSoft en este repo | Demo vertical | Hay una vertical demo integrada en la web | Que aquí vive el producto B2B completo |
| Comparación entre repositorios | Vision | Es una dirección estratégica coherente con la filosofía del proyecto | Que ya compara repos de forma continua y operativa |
| Comunidad de agentes coordinados | Vision | Es la categoría que el proyecto quiere fundar | Que hoy ya está cerrada, escalada y probada |
| Trazabilidad y auditoría | Defendible | El proyecto valora control, persistencia y evidencia, y puede estructurarse hacia auditoría fuerte | Que toda la cadena de auditoría de la nueva etapa ya está implementada |
| Modo Fundador | Roadmap | Puede articularse como acceso temprano con influencia real | Que ya existe como programa operativo completo |

## Qué existe hoy

- motor Python ejecutable localmente
- documentación extensa
- demo web funcional
- verticales demo dentro del repo
- primera capa de IA local utilitaria
- cultura explícita de trazabilidad y separación entre producto y demo

## Qué es defendible públicamente

- Artificial World ya tiene una base real, no es una idea vacía
- la dirección local-first y open source encaja con la naturaleza del proyecto
- la evolución hacia comprensión auditable de proyectos es coherente con su filosofía
- la trazabilidad debe ser un rasgo del producto, no un detalle de marketing

## Qué sigue siendo visión

- comparación iterativa entre repos propios y de referencia
- MCP maestros coordinando agentes especializados
- expediente completo de proyecto con contradicciones versionadas
- experiencia fundador estructurada
- comunidades abiertas de agentes y perfiles de trabajo

## Claims recomendados

- sistema local y open source
- base real de simulación y memoria
- dirección auditable y trazable
- evolución hacia comprensión útil de proyectos reales
- arquitectura orientada a evidencia

## Claims prohibidos por ahora

- comprende cualquier repo perfectamente
- reemplaza arquitectos o equipos
- detecta todos los problemas automáticamente
- coordina inteligencias en tiempo real a escala probada
- ya es el estándar del mercado

## Test de publicación

Antes de publicar algo, comprobar:

1. ¿Esto existe hoy o describe una dirección futura?
2. ¿Se puede demostrar con código, documentación o prueba?
3. ¿Una lectura hostil lo vería como riguroso o como humo?
4. ¿Está separado con claridad de demo, visión y roadmap?

Si alguna respuesta falla, la frase debe reescribirse.
